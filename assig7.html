Assignment 7: BEFORE Triggers

📋 Problem Statement

📖 Library Audit System

Create triggers to automatically log UPDATE and DELETE operations on Library table, maintaining a complete audit trail in Library_audit table.

1️⃣ Database Setup

Step 1 
USE te31145_db;

CREATE TABLE Library (
    book_id INT PRIMARY KEY NOT NULL,
    book_name VARCHAR(100),
    author VARCHAR(50),
    copies INT
);

CREATE TABLE Library_audit (
    audit_id INT AUTO_INCREMENT PRIMARY KEY,
    book_id INT,
    book_name VARCHAR(100),
    author VARCHAR(50),
    copies INT,
    action_type VARCHAR(10),
    action_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
💡 Design Notes:

audit_id: AUTO_INCREMENT for unique record ID

action_type: Stores "UPDATE" or "DELETE"

action_time: Automatically records timestamp

2️⃣ BEFORE UPDATE Trigger

Step 2 
DELIMITER $$

CREATE TRIGGER before_library_update
BEFORE UPDATE ON Library
FOR EACH ROW
BEGIN
    INSERT INTO Library_audit(book_id, book_name, author, copies, action_type)
    VALUES (OLD.book_id, OLD.book_name, OLD.author, OLD.copies, "UPDATE");
END $$

DELIMITER ;
How it works:

Fires automatically before any UPDATE on Library table and captures OLD values before they're overwritten.

3️⃣ BEFORE DELETE Trigger

Step 3 
DELIMITER $$

CREATE TRIGGER before_library_delete
BEFORE DELETE ON Library
FOR EACH ROW
BEGIN
    INSERT INTO Library_audit(book_id, book_name, author, copies, action_type)
    VALUES (OLD.book_id, OLD.book_name, OLD.author, OLD.copies, "DELETE");
END $$

DELIMITER ;
How it works:

Fires automatically before DELETE and preserves the deleted record in audit table.

💡 Key Concepts

1. OLD and NEW Keywords

Operation OLD NEW
INSERT Not Available New row values
UPDATE Before update After update
DELETE Deleted row Not Available

2. BEFORE vs AFTER Triggers

BEFORE: Execute before DML operation - can access/modify data before change

AFTER: Execute after DML operation - operation already completed

Our Use: BEFORE triggers capture original values before they're lost

3. FOR EACH ROW

Trigger fires once for each affected row. If UPDATE affects 5 rows, trigger executes 5 times.

🛠️ Useful Commands 

-- View all triggers
SHOW TRIGGERS;

-- View trigger definition
SHOW CREATE TRIGGER before_library_update;

-- Drop trigger
DROP TRIGGER IF EXISTS before_library_update;

-- Count audit records by type
SELECT action_type, COUNT(*) as count
FROM Library_audit
GROUP BY action_type;
