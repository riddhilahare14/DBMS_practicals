Assignment 2: Basic SQL Operations

📋 Table Schema

Database Structure

We're working with 3 interconnected tables:

  • Dept: Department information
  • Employee: Employee details linked to departments
  • Project: Project information linked to departments
USE te31145_db;

CREATE TABLE Dept (
    dept_id INT PRIMARY KEY,
    dept_name VARCHAR(50),
    dept_location VARCHAR(20)
);

CREATE TABLE Project (
    proj_id INT PRIMARY KEY,
    dept_id INT,
    proj_name VARCHAR(20),
    proj_location VARCHAR(20),
    proj_cost INT,
    proj_year INT,
    FOREIGN KEY (dept_id) REFERENCES Dept(dept_id) ON DELETE CASCADE
);

CREATE TABLE Employee (
    emp_id INT PRIMARY KEY,
    dept_id INT,
    emp_fname VARCHAR(20),
    emp_lname VARCHAR(20),
    emp_position VARCHAR(50),
    emp_salary INT,
    emp_joinDate DATE,
    FOREIGN KEY (dept_id) REFERENCES Dept(dept_id) ON DELETE CASCADE
);

2 Filter Employees with Multiple Conditions

💡 Concepts Used

JOIN: Combines Employee and Dept tables

LIKE: Pattern matching (% means "anything")

WHERE: Filters data based on conditions

Method I: Using OR 
SELECT * FROM Employee e
JOIN Dept d ON d.dept_id = e.dept_id
WHERE (d.dept_name = 'Computer' OR d.dept_name = 'IT') 
AND (e.emp_fname LIKE 'p%' OR e.emp_fname LIKE 'h%');
Method II: Using IN 
SELECT * FROM Employee e
JOIN Dept d ON d.dept_id = e.dept_id
WHERE d.dept_name IN ('Computer', 'IT') 
AND (e.emp_fname LIKE 'p%' OR e.emp_fname LIKE 'h%');
In simple words: "Show me all employees from Computer or IT department whose first name starts with P or H"

3 List Unique Employee Positions

💡 Concept: DISTINCT

DISTINCT removes duplicates - like getting a list of unique items from your shopping cart.

SELECT DISTINCT emp_position FROM Employee;

-- Alternative: Count unique positions
SELECT COUNT(DISTINCT emp_position) AS total_positions FROM Employee;
Result: Shows each job position only once (Developer, Analyst, HR Manager, etc.)

4 Salary Increase for Employees Joined Before 1985

💡 Concept: UPDATE Statement

UPDATE modifies existing data. The YEAR() function extracts the year from a date.

UPDATE Employee
SET emp_salary = emp_salary * 1.1    -- 10% increase
WHERE YEAR(emp_joinDate) < 1985;
Affected Employees: Peter Parker, Harry Potter, Pamela Anderson, Helen Mirren, Hannah Montana (5 employees)

5 Delete Departments in Mumbai

💡 Concept: DELETE with CASCADE

DELETE removes rows. CASCADE means related records in other tables are automatically deleted too.

DELETE FROM Dept
WHERE dept_location = 'Mumbai';
⚠️ Warning: Deletes Computer & Marketing departments AND all their employees and projects!

7 Projects with Cost Between 100000 and 500000

💡 Concept: BETWEEN

BETWEEN finds values in a range (inclusive of both endpoints).

-- Method 1: Using BETWEEN
SELECT * FROM Project WHERE proj_cost BETWEEN 100000 AND 500000;

-- Method 2: Using AND
SELECT * FROM Project WHERE proj_cost >= 100000 AND proj_cost <= 500000;

8 Maximum and Average Project Cost

💡 Concept: Aggregate Functions

MAX(): Finds highest value | AVG(): Calculates average

-- Find most expensive project
SELECT * FROM Project 
WHERE proj_cost = (SELECT MAX(proj_cost) FROM Project);

-- Calculate average cost
SELECT AVG(proj_cost) AS avg_proj_cost FROM Project;
Subquery: The inner query (in parentheses) runs first and returns the max cost, then the outer query uses that value.

9 Sort Employees by Last Name (Descending)

💡 Concept: ORDER BY

DESC: Descending order (Z to A) | ASC: Ascending order (default)

SELECT emp_id, emp_fname, emp_lname
FROM Employee
ORDER BY emp_lname DESC;

10 Projects Started in 2004, 2005, or 2007

💡 Concept: IN Operator

IN checks if a value matches any item in a list (cleaner than multiple OR conditions).

SELECT proj_name, proj_location, proj_cost
FROM Project
WHERE proj_year IN (2004, 2005, 2007);
Equivalent to: WHERE proj_year = 2004 OR proj_year = 2005 OR proj_year = 2007
